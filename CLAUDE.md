# UnLeaf Project (v1.00 C++ Native) - プロジェクト憲法

## セッション・ドキュメント管理
- **本ファイルは本プロジェクトにおける最上位の判断基準であり、他のいかなる設計資料やコメントよりも優先される。
- **文脈の再構築**: 起動時、または `/clear` / `/compact` 等で履歴がリセットされた直後の最初のターンでは、自律的に `SESSION_CONTEXT.md` を読み込み、プロジェクトの最新状況を再把握すること。
- **実行計画の提示**: コード修正前には必ず「Plan（実行計画）」を提示し、ユーザーの承認を得ること。
- **現状上書き型同期**: 仕様変更・UI刷新・技術決定の後は、`SESSION_CONTEXT.md` を「現状上書き型」で整理・更新すること（古い情報は削除する）。
- **要約報告**: 更新完了後、何を変更し、何を削除したかを簡潔にユーザーに報告すること。
- **本プロジェクトには補助資料として SESSION_CONTEXT.md および .claude 配下の設計資料が存在するが、判断が衝突した場合は本ファイルを最優先とする。
- 方針や実装の着手について逐一確認を求めないこと
- 自信がある場合はそのまま実装に進むこと
- 判断に迷う場合や、破壊的な変更の場合のみ確認すること
- 複数の選択肢がある場合は、最も妥当なものを選んで進め、
  選択理由を作業後に簡潔に報告すること

## バージョン管理ルール
- types.h の VERSION 定数の変更（バージョンアップ）は、ユーザーの明示的な指示がない限り禁止。
- ログ・ライブログへのバージョン情報の追加は、ユーザーの明示的な指示がない限り禁止。
- ログ出力文字列にバージョンマーカー（例: `[v6.0]`, `v6.0` 等）を含めることを禁止する。機能カテゴリタグ（例: `[REGISTRY]`）を使用すること。
